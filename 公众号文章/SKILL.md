# AI对话提炼成公众号文章

## 适用场景

用户是一位从业 10 年的互联网产品经理，与 AI 进行多轮对话，通过追问深挖获得有价值的洞察，希望将这些认知沉淀成一篇微信公众号文章。读者主要是用户的同事、同行和日常接触的业内人士。公众号的定位不是营销涨粉，而是展现作者的专业深度和独立思考能力——让读者觉得"这个人对这件事想得很透"。

AI 对话是素材来源，不是文章结构。输出的文章应该呈现为「作者自己深度思考后的认知沉淀」，而不是「整理 AI 对话的记录」。读者不需要知道这篇文章来自 AI 对话。

## 输入

- 原始对话记录（完整复制粘贴即可）

## 输出物

### 目录结构

在素材文件所在的目录下，创建一个以文章主题命名的文件夹（中文，简明概括，如 `AI正向飞轮`、`产品经理能力模型`），然后：
1. 将原始素材文件移动到该文件夹内
2. 在该文件夹内生成以下三个文件

| 文件 | 文件名 | 内容 |
|------|--------|------|
| 完整文章 | `文章.md` | 整理后的公众号正文 |
| 标题与简介 | `标题与简介.md` | 50个备选标题 + 10个简介版本 |
| 配图提示词 | `配图提示词.md` | 10组配图生成提示词 |

示例：素材文件为 `公众号素材/某次AI对话.md`，文章主题为"AI正向飞轮"，则最终结构为：
```
公众号素材/AI正向飞轮/
  ├── 某次AI对话.md      ← 原始素材（移入）
  ├── 文章.md
  ├── 标题与简介.md
  └── 配图提示词.md
```

## 重要：分阶段执行规则

本 Skill 必须分 4 个阶段执行，每个阶段完成后必须停下来等用户确认再继续。严禁一次性生成所有内容。

- 阶段一：分析素材 → 输出内容地图和审查结果 → 等用户确认
- 阶段二：撰写文章 → 输出 `文章.md` → 等用户确认
- 阶段三+四：生成标题与简介 → 输出 `标题与简介.md` → 直接继续生成配图提示词 → 输出 `配图提示词.md`

阶段一、阶段二结束时，必须停下来等用户确认再继续。阶段三完成后直接执行阶段四，无需等待用户确认。

## 核心原则

### 1. 内容筛选：只保留干货

保留：具体的方法论/框架/模型、有启发性的观点和洞察、可复用的经验或案例、能引发思考的对比和反思

删除：寒暄客套话、AI的过度解释或重复表达、基础常识性内容、与主题无关的闲聊

### 2. 追问处理：保留价值，包装表达

有价值的追问（引出新角度、深化理解、获得案例/方法论）→ 保留并包装
无价值的追问（基础概念澄清、重复确认）→ 删除

包装原则：把「我不懂」转化为「这里有个关键点值得深挖」，保留追问的思考路径，但用更专业的表达

### 3. 叙事视角：作者第一视角（默认）

默认以作者第一视角写作，完全抛开对话结构。AI 不出现，或极少出现。文章读起来是「我思考了这个问题，得出了这些判断」。

例外：当对话本身有戏剧性或洞察性时，可以主动选择「采访体」，把 AI 的回答当作一个有见地的受访者来呈现。这是主动选择，不是默认。

禁止：「我问 AI：……」「AI 的回答让我意识到：……」、全篇问答对话结构

### 4. 犀利写作：观点必须有锋芒

文章的核心竞争力是犀利的判断，不是面面俱到的描述。读者关注的不是你知道多少，而是你敢不敢下判断、判断是否精准。

观点表达原则：
- 每个核心论点必须包含一个明确的、有攻击性的判断，不能只描述机制或现象。「正反馈会强化行为」是描述，「问题太轻，飞轮根本没有被推动的机会」是判断
- 直接点名大多数人在做什么错事，不要用「有些人可能……」这种模糊表达。「说白了，AI 在他们手里就是一个不会顶嘴的搜索引擎」比「很多人可能没有充分利用 AI」强十倍
- 允许让读者不舒服。好文章的标准不是「读完觉得有道理」，而是「读完被刺到了」

案例构建原则：
- 正面案例要有创造性和冲击力：不写「AI 帮我提高了效率」，写「完全不懂代码，却让 AI 从零做出了一个能跑的小游戏」
- 反面案例要直接揭示错误行为：不写「使用 AI 时要注意判断」，写「让 AI 生成一套工作流，以为拿来就能用——如果你对这个东西没有判断力，没有叠加你自己的审美和要求，那它就是概率分布里采样出来的通用模板」
- 每个论点的案例要同时覆盖「做对了是什么样」和「做错了是什么样」，形成对比张力

结构犀利化：
- 开头直接判断读者的现状，不要从背景或趋势入手。「用了一年 AI，很多人发现自己仍然停在同一个水平线上」比「AI 正在改变我们的工作方式」有力得多
- 收尾用二元判断而非温和邀请。「到了这个阶段的人，不需要任何人告诉他要拥抱 AI。而没到这个阶段的人，再多的培训课和提示词模板也救不了他」比「希望大家都能找到自己的节奏」犀利得多
- 禁止使用弱比喻收尾（如「就像骑自行车一样」），结尾要么是一个让人无法反驳的判断，要么是一个让人后背发凉的事实
- 章节标题用叙事句而非学术标签。「真正的燃料是正反馈」比「正反馈机制」有力；「转起来就回不去了」比「飞轮的不可逆性」有画面感

试金石判断：
- 文章中至少包含 2-3 个「试金石」式的判断标准，让读者可以立刻对号入座、无法回避。例如：「判断一个人 AI 用得好不好，看他是不是在用语音输入」「还在逐字敲键盘写提示词的人，大概率没跨过那个临界点」
- 这类判断要足够具体、足够尖锐，让读者读完必须做出「我是/我不是」的自我归类

语气控制：
- 犀利不等于刻薄。判断要精准，不要人身攻击
- 犀利不等于煽动。用事实和逻辑支撑判断，不要靠情绪词堆砌
- 最高级的犀利是「说完一句话，对方沉默了」——不需要感叹号，不需要反复强调，一句话切到要害

## 文章结构模板

根据对话内容选择合适的结构，不要强行套用固定模板：

### 模板 D：认知沉淀型（默认推荐）

适用于：大多数多轮对话，作者从中形成了自己的判断

```
开头：一个让人有共鸣的场景或问题
→ 核心观点（作者自己的判断）
→ 支撑逻辑（多条线索融合后的论证）
→ 反转或深化（让读者有「原来如此」的感觉）
→ 结尾：行动方向或开放性思考
```

### 模板 A：问题驱动型

适用于：围绕一个具体问题展开的对话

```
开头：问题背景 + 为什么重要
→ 核心洞察1 + 案例支撑
→ 追问深挖 + 更深层理解
→ 核心洞察2 + 方法论
→ 结尾：总结 + 行动建议
```

### 模板 B：认知升级型

适用于：通过对话改变了原有认知

```
开头：我原本以为是 A
→ 对话过程中的发现
→ 原来真相是 B
→ 为什么会这样（底层逻辑）
→ 结尾：新的行动方向
```

### 模板 C：方法拆解型

适用于：从对话中提炼出可复用的方法

```
开头：遇到的问题场景
→ 方法的核心思路
→ 具体步骤拆解
→ 案例演示
→ 结尾：适用场景和注意事项
```

## 格式规范

### 文章格式

正文只允许：`#` / `##` / `###` 标题、纯文本段落

严格禁止：`**加粗**`、`- 列表` / `1. 有序列表`、`> 引用块`、`---` 分隔线、括号里的英文术语

篇幅控制：
- 默认正文控制在 3000 字以内（含标题）
- 素材信息量确实很大时，允许突破到 4500 字上限，但需要写作前向用户说明并确认
- 初稿超长时，优先压缩低信息密度段落，而不是砍掉关键例子

段落结构：
- 轻阅读风格：段落拆短，每段2-4句话，保持快节奏，降低阅读压力
- 每个段落只承载一个信息点或一个论述步骤，不要在一个段落里塞多层逻辑
- 段落之间靠逻辑递进自然衔接，不需要过渡句

语言精简：
- 删减不必要的虚词（"的""地""了""着"），只在影响语义理解时保留
- 去掉无信息量的修饰词和连接词（"其实""确实""非常""比较""还是""当然"等）
- 去掉冗余的过渡句，段落之间靠逻辑关系自然衔接，不需要"接下来""下面""让我们"等引导语
- 能用短句说清楚的不用长句，能用一个词的不用一个短语

空行规则（为方便直接复制到公众号编辑器）：
- 标题下面直接接正文，不空行
- 段落与段落之间不空行
- 每个章节的最后一段与下一个标题之间，保留一个空行

排版规则：
- 中英文混排时，英文单词前后各加一个半角空格（如"使用 Claude Code 进行开发"）
- 中文与数字混排时，数字前后各加一个半角空格（如"用户量停留在 100 万级别"）
- 标点符号紧跟前文，不受此规则影响（如"有 3 个原因："）

标点规则：
- 每句话或每个段落的末尾不加句号
- 其他标点正常使用（逗号、问号、感叹号、冒号、引号等）

其他：保持专业语气，减少个人情绪词

### 标题风格

标题关键词提取（在生成标题前必须先完成此步骤）：
- 从文章中提取 8-12 个高辨识度的关键词/短语，分为三类：
  - 核心概念词：文章独有的模型名称、方法论、术语（如"五常军费模型""零基预算""中层管理挤压"）
  - 关键数字：文中出现的具体金额、比例、倍数（如"200 美元""30 倍杠杆""2% 及格线"）
  - 具象场景词：能唤起画面感的动作或场景（如"深夜下单""算完账""打开付费页面"）
- 这些关键词是标题的核心素材。10 种心理触发维度只是拼接方式，关键词才是用户扫一眼就被抓住的东西
- 每个标题必须至少包含 1 个关键词。好的标题通常包含 2 个（一个概念词 + 一个数字，或一个场景词 + 一个概念词）

标题通用规则：
- 字数控制在 15-25 字，前 15 字放最核心的信息（手机推送截断线，前 15 字决定生死）
- 数字一律用阿拉伯数字（"3 个方法"而非"三个方法"），数字在视觉上天然突出
- 关键词前置：最抓人的概念词、数字、场景词放在标题最前面，不要埋在后半段
- 标点策略：问号制造悬念，省略号留白想象，逗号做节奏分隔；禁用感叹号（显得廉价）
- 每个标题必须通过"扫一眼测试"：读者在信息流里 0.5 秒扫过，能不能被抓住

50 个标题，覆盖以下 10 种心理触发维度（每种 5 个）。每种维度下方提供了句式模板作为参考骨架，但同一维度内的 5 个标题必须在句式、节奏、长短、切入角度上充分变化，严禁套用同一模板批量填词：

1. 悬念好奇型 — 说一半留一半，制造"不点开就不舒服"的信息缺口
   参考句式：
   - [某件事/某个发现]，[悬念词：原因竟然是……/真相是……]
   - [看似普通的事] + 背后的真相/秘密
   - [某个结论]，但很少有人注意到 [留白]

2. 故事场景型 — 用具体场景、人物、时间线制造代入感
   参考句式：
   - [人物/身份] + [起点状态]，[转折/逆袭结果]
   - [具体时间/场景] + [发生了什么]，[结果出人意料]
   - 从 [A 状态] 到 [B 状态]，[人物] 只做了 [一件事/N 步]

3. 对比冲突型 — 用认知反差、两极对照制造张力
   参考句式：
   - [A] 和 [B] 的差距，就在 [核心变量]
   - 以为是 [A]，其实是 [B]
   - [同一件事]，[高手/普通人] 的做法完全相反

4. 干货价值型 — 明确的价值承诺，让读者一眼看到"点开能得到什么"
   参考句式：
   - [数字] 个 [方法/技巧/原则]，[可量化的好处]
   - 一篇讲透 [主题]（建议收藏）
   - [痛点场景]？[数字] 个方法 + [具体好处]

5. 反常识型 — 颠覆已有认知，用反直觉结论引发好奇
   参考句式：
   - 我劝你别 [看似正确的做法]，除非你想 [负面后果]
   - [大多数人相信的观点]，其实是错的
   - [某个领域] 最大的误解：[颠覆性判断]

6. 否定警示型 — 用"别再""停止""不要"制造紧迫感和行动冲动
   参考句式：
   - 别再 [某个常见做法] 了，[负面后果]
   - [某类人] 最容易犯的错：[具体行为]
   - 如果你还在 [某个做法]，趁早停下来

7. 身份共鸣型 — 点出目标人群的身份或处境，让读者觉得"说的就是我"
   参考句式：
   - [职业/身份] + [年限/阶段]，最该明白的 [数字] 件事
   - 工作 [N 年] 还在 [某个困境]？问题出在 [核心原因]
   - 给 [特定人群] 的 [数字] 条建议

8. 情绪共振型 — 戳中情绪体验，引发"被说中了"的共鸣
   参考句式：
   - [某类人] 的 [情绪/困境]，都是从 [具体原因] 开始的
   - [年龄/阶段] 以后才明白，[一个扎心的道理]
   - [一个普遍但没人说破的感受]

9. 权威背书型 — 借助权威来源增强可信度和说服力
   参考句式：
   - [权威人物/机构] + [颠覆认知或高价值的结论]
   - [知名公司] 内部的 [方法/文档/原则]
   - [领域专家] 提醒：[大多数人忽略的事]
   注意：仅在文章确实引用了权威来源时使用，不可捏造

10. 结果展示型 — 用具体的、可量化的结果倒推吸引力
    参考句式：
    - 用了 [方法]，[时间] 内 [具体可量化结果]
    - 坚持 [行为] + [时间]，[结果变化]
    - [一个动作/决定]，让 [某个指标] 翻了 [倍数]

组合技巧（鼓励跨维度叠加，效果更强）：
- 痛点 + 数字 + 好奇心：把读者的困境、具体数字、未揭示的答案组合在一起
- 权威 + 反常识 + 利益：借权威之口说出颠覆性结论，同时暗示读者能获得什么
- 故事 + 数字 + 结果：用人物起点、具体数字、逆袭结果构成完整叙事弧
- 身份 + 情绪 + 悬念：先锁定人群，再戳中情绪，最后留一个未解答的问题

多样性强制要求（50 个标题的防单调规则）：
- 句式打散：同一维度内的 5 个标题，至少使用 3 种不同的句式结构，禁止连续 2 个标题用相同句式
- 长短交替：每组 5 个标题中，至少包含 1 个短标题（15 字以内）和 1 个长标题（20 字以上）
- 开头词去重：50 个标题中，同一个开头词（如"为什么""如何""别再"）最多出现 3 次
- 语气混搭：陈述句、疑问句、祈使句、感叹句（不带感叹号的那种）交替使用，不要全是疑问句或全是陈述句
- 视角轮换：有的标题站在作者视角（"我发现……"），有的站在读者视角（"你可能不知道……"），有的用第三人称（"那些……的人"），不要锁死在一个视角
- 情绪光谱：不要全是焦虑向的标题，也要有好奇、向往、幽默、温暖等正向情绪的标题穿插其中
- 关键词分配：提取的 8-12 个关键词要均匀分布在 50 个标题中，不要前 20 个标题把关键词用完，后面全是空泛表达

标题避坑清单（生成后逐条检查）：
- 禁止论文式标题："浅谈……""论……的重要性""关于……的思考"
- 禁止自嗨式标题：只站在作者角度，读者看完无感、无法代入
- 禁止"看完标题就够了"：标题已经把结论说完，读者没有点开的动力
- 禁止低质标题党词汇："震惊""不转不是中国人""速看马上删""太可怕了"
- 禁止空洞修饰：堆砌"最全""最强""史上最""终极"等没有实质内容的词
- 检查信息缺口：读完标题后，读者是否还有"想知道答案"的冲动？如果没有，这个标题不合格

### 简介规范

简介（摘要）是标题之外的第二次说服机会，出现在订阅号消息列表、转发卡片、朋友圈分享、搜一搜结果等位置。

字数与截断：
- 后台上限 120 字，但实际展示通常只显示前 30-50 字
- 最佳长度 50-80 字，关键信息必须在前 30 字内出现
- 必须手动填写，禁止依赖系统自动抓取（系统默认抓正文前 54 字，通常是废话）

核心原则：简介和标题是组合拳，不是各自为战
- 简介必须提供标题之外的信息增量，禁止重复标题已说过的内容
- 标题负责"抓眼球"，简介负责"给理由"——告诉读者为什么值得点开
- 让读者知道"点进去能获得什么"，建立明确的阅读预期

10 个版本，覆盖以下 10 种写法（每种 1 个）：

1. 价值承诺型 — 直接告诉读者能获得什么
   示例：3 个万能模板 + 12 个高频场景话术，拿来就能用。附完整案例

2. 悬念好奇型 — 制造信息缺口，激发好奇心
   示例：不靠打折、不靠加盟，只用了一个大多数人忽略的方法

3. 痛点共鸣型 — 先戳中痛点，再暗示有解决方案
   示例：问题可能不在内容，而在结构。改掉这 3 个习惯，通过率翻倍

4. 数据冲击型 — 用具体数字或反常识事实吸引注意
   示例：分析了 1000+ 品牌数据，发现 5 个反直觉的变化。第 3 个可能颠覆你的认知

5. 场景代入型 — 描述具体场景，让读者产生"说的就是我"的感觉
   示例：如果你也经历过"开了 2 小时会，结论是再开一次"，这篇一定要看

6. 提问引导型 — 用问句引发思考，激发点击欲望
   示例：你的私域为什么越做越累？可能从第一步就走错了

7. 干货预告型 — 明确告知内容结构和含金量
   示例：从账号定位到爆款笔记，8000 字长文，建议收藏慢慢看

8. 结论先行型 — 抛出核心结论，让读者想知道为什么
   示例：真正拉开差距的不是能力，是对"无聊工作"的态度

9. 反常识型 — 用颠覆认知的判断制造好奇
   示例：越努力越焦虑，不是你不够好，是方向选错了

10. 情绪共振型 — 戳中情绪体验，引发"被说中了"的共鸣
    示例：那些看起来毫不费力的人，只是把崩溃的时刻藏起来了

简介避坑清单：
- 禁止重复标题：标题说了"3 个方法"，简介不要再说"本文介绍 3 个方法"，应补充"亲测有效，第 2 个最容易上手"
- 禁止过于笼统："本文介绍了一些方法"这种没有信息量的描述
- 禁止过度夸张："看完人生开挂""不看后悔一辈子"会降低信任感
- 禁止信息过载：不要在 80 字里塞太多内容，抓住一个点打透
- 禁止与内容不符：简介承诺的内容文章里必须有，否则损害长期信任

### 配图提示词规范

10 组配图提示词，分为两类：5 组封面图 + 5 组文内配图。每组提供一个完整的中文 prompt + 中文说明。用户会在即梦等 AI 图片生成工具中使用这些提示词，每组生成多张后挑选。

#### 通用风格要求（封面图和文内配图共用）

画面必须具有单一且明确的具体视觉主体（人物、物体或动物），讲述一个富有想象力但绝非抽象的故事场景。艺术风格应当尽可能随机和多样化（鼓励尝试各种不同的视觉表现形式），但无论何种风格，都必须服从于温暖、明亮、高饱和度的整体基调，以及柔和舒适的光影氛围。构图必须简洁大胆，确保在手机屏幕上清晰醒目，且画面必须是一个完整的统一体。严禁科幻机械主题、机器人形象、带有科技感的视觉载体、抽象光影色块、低质感概念草图、深色调画面。严禁出现文字、字母或乱码符号。杜绝物体融合、背景堆砌、反重力悬浮、无尽阶梯等物理逻辑错误。拒绝塑料质感、过度平滑与油腻光泽。

颜色禁忌：避免纯白背景（深色模式下刺眼）和灰暗低饱和度配色（信息流中容易被忽略）。优先使用高对比度配色（互补色组合如橙+蓝、黄+紫）或暖色调为主的方案。

人物要求：当画面中出现人物时，优先使用女性角色。女性角色应当是有吸引力的、风格多样的（可以是不同年龄段、不同穿着风格、不同文化背景的美女），但必须自然真实，不能是刻板的模特姿态。人物应当有动作、有表情、有故事感，而不是摆拍。

场景吸引力原则：人物必须置于有想象力的、超现实的或奇幻的场景中，而不是日常写实场景。"女性坐在书桌前看电脑""女性在咖啡馆喝咖啡""女性在海边跑步"这类纯写实日常场景缺乏视觉吸引力，必须避免。好的做法是让人物与超出日常经验的元素互动——场景应当包含不可能在现实中看到的事物、尺度、材质或空间关系，但必须紧扣文章主题而非随意堆砌奇幻元素。场景越超出日常经验，画面越有吸引力。判断标准：如果把画面描述念给别人听，对方会不会说"我想看看这张图"——如果不会，就需要加入更多想象力。

物品禁忌：禁止把单一物品（天平、金币、算盘、灯泡、齿轮、沙漏等）放大为画面主体。这类"概念图标放大"的构图缺乏故事性和吸引力。如果需要表达抽象概念，必须通过人物的动作、场景的氛围、环境的细节来间接传达，而不是直接展示一个巨大的象征物。好的配图应该让人想问"这个人在做什么？发生了什么事？"，而不是"这是什么东西？"。

#### 封面图专属要求（前 5 组提示词）

封面图是影响打开率最大的视觉元素，在信息流缩略图状态下必须抓人。

安全区域（核心）：
- 封面图在朋友圈分享时会被裁成 1:1 正方形，只保留画面中央区域
- 因此核心视觉主体必须集中在画面正中央，左右两侧只放装饰性内容
- 提示词中必须包含"主体居中构图，核心元素集中在画面中央"的描述

缩略图可读性：
- 封面在手机信息流中显示很小，画面元素必须足够大、足够简洁
- 禁止细碎元素、密集纹理、微小细节——缩小后全部变成模糊色块
- 主体占画面面积不低于 40%，背景尽量干净
- 提示词中加入"构图极简，主体突出，背景干净"等约束

封面图与标题的配合：
- 封面图应与标题形成互补而非重复——标题说"是什么"，封面展示"什么感觉"
- 5 组封面图提示词应分别对应不同的视觉策略，从以下 5 种中选择（也可根据文章主题灵活调整）：
  1. 情绪氛围型 — 用色彩、光影、空间营造文章的核心情绪
  2. 具象场景型 — 用一个具体的、有故事感的场景传达文章主题
  3. 隐喻象征型 — 用视觉隐喻间接表达文章的核心概念
  4. 人物特写型 — 以人物的表情、动作、姿态为核心，传递情感张力
  5. 奇幻叙事型 — 用超现实的场景构建一个让人想点进去的视觉悬念

#### 文内配图专属要求（后 5 组提示词）

文内配图是读者已经点进文章后看到的，可以比封面图更有细节和想象力。

多样性要求：5 组提示词必须使用 5 种不同的视觉主体和场景，禁止多张图围绕同一个具象物件反复变换角度或材质。每组提示词应从文章的不同侧面（核心隐喻、具体场景、情绪感受、人物动作等）出发，确保视觉上互不重复。

5 种视觉思路模板（根据文章主题灵活选择 5 种）：

1. 超现实隐喻型 — 把核心概念转化为超现实视觉隐喻场景
2. 微观放大型 — 把日常事物放大到不寻常的尺度
3. 空间错位型 — 把不该出现在一起的元素放在同一空间
4. 视角颠覆型 — 用极端视角重新看待熟悉场景
5. 材质变异型 — 让熟悉物体用不可能的材质呈现
6. 时间叠加型 — 同一画面展示不同时间状态的叠加
7. 情绪景观型 — 把情绪或感受转化为风景/空间
8. 对比融合型 — 画面中两种对比元素（风格、质感、状态）自然共存于同一场景，通过柔和渐变或有机过渡形成张力，禁止从中间一分为二的分割构图

## 执行步骤

### 阶段一：分析素材（输出后暂停）

1. 阅读原始对话
   - 完整阅读，标记核心洞察点
   - 识别需要删除的「水」内容
   - 识别有价值追问 vs 需删除的追问

2. 绘制内容地图
   - 列出对话中所有洞察点
   - 找出它们之间的逻辑关系：并列、递进、因果、对比
   - 根据逻辑关系决定文章结构，而不是按对话顺序排列
   - 标记关键概念和精华例子：核心术语、精准定义、生动类比、具体场景举例，必须逐一标记，确保成文时保留

3. 素材审查：识别错漏与争议
   - 审查维度：事实性错误、逻辑漏洞、过度简化、争议性观点
   - 发现问题时逐条列出：问题内容 → 问题类型 → 原因 → 建议处理方式
   - 没有明显问题则简要说明审查结论

4. 确定文章结构
   - 优先考虑模板 D（认知沉淀型），根据内容地图选择最合适的模板
   - 核心洞察前置：最有力的洞察应尽早亮出来，不要按「先现象后本质」的惯性排列

⏸️ 暂停点：将内容地图、审查结果、推荐的文章结构展示给用户。如有审查问题，等用户逐条确认。用户确认后再进入阶段二。

### 阶段二：撰写文章（输出后暂停）

5. 撰写文章初稿
   - 以作者第一视角写作，AI 不出现或极少出现
   - 多条线索融合论证，不按对话顺序平铺
   - 确保内容密度高，每段都有信息量
   - 保留关键概念的原始表述（核心术语、精准定义不能用模糊概括替代）
   - 保留有说服力的例子（具体场景举例、对比演示、正反案例是说服力核心）
   - 每个核心论点至少配一个具体例子
   - 用视觉化类比替代抽象解释
   - 允许立场和棱角，不追求面面俱到
   - 犀利写作（参见核心原则第 4 条）：
     - 开头直接判断读者现状，不从背景趋势入手
     - 每个论点必须包含明确判断，不能只描述机制
     - 案例要同时覆盖「做对了」和「做错了」的对比
     - 章节标题用叙事句，不用学术标签
     - 收尾用二元判断，禁止弱比喻和温和邀请
     - 至少包含 2-3 个试金石式判断标准
     - 密集分析段落要拆短，每段 2-4 句，保持节奏感

6. 逐句精简审查（写完初稿后、输出文件前的强制步骤）
   - 逐段扫描，删除以下冗余：
     - 无信息量的虚词："的""地""了""着"，只在影响语义理解时保留
     - 口水连接词："其实""确实""非常""比较""还是""当然""然后""所以说"
     - 冗余过渡句："接下来""下面""让我们""我们来看看"
     - 重复修饰：同一个意思用不同说法说了两遍的，只保留更精准的一个
     - 可删除的代词和指示词："这个""那个""它的""我们的"，上下文已明确时直接删
   - 能用短句说清楚的不用长句，能用一个词的不用一个短语
   - 句末句号清除：逐段检查，删除每句话和每个段落末尾的句号（"。"）。这是硬性格式要求，不是风格偏好。逗号、问号、感叹号、冒号等其他标点不受影响
   - 精简后全文字数应比初稿减少 5%-10%

7. 清理格式
   - 删除所有加粗、列表、引用块、分隔线、括号里的英文术语
   - 按空行规则检查：标题下不空行、段落间不空行、章节末尾与下一标题间保留一个空行
   - 句号最终检查：确认全文没有任何句末句号（"。"），这是最容易遗漏的格式问题
   - 检查是否有「AI 说」「我问 AI」等暴露来源的表达

8. 输出到 `文章.md`，文件开头附带以下元信息（不算入正文字数）：
   ```
   <!--
   字数：XXXX 字
   主线：一句话概括文章叙事弧线
   段落结构：
   - 章节标题1：一句话概括
   - 章节标题2：一句话概括
   - ...
   -->
   ```

⏸️ 暂停点：告诉用户文章已完成，请用户阅读 `文章.md` 并反馈。用户确认满意后再进入阶段三。如用户有修改意见，先修改文章直到用户满意。

### 阶段三：生成标题与简介（输出后暂停）

9. 生成标题与简介，输出到 `标题与简介.md`
   - 50 个备选标题，按 10 种心理触发维度分组，每种 5 个
   - 10 个简介版本，每个 50-80 字，覆盖 10 种写法

阶段三完成后直接继续阶段四，无需暂停。

### 阶段四：生成配图提示词

10. 生成配图提示词，输出到 `配图提示词.md`
   - 根据文章主题，从8种视觉思路模板中选择最契合的思路
   - 生成10组完整的中文 prompt（5组封面图 + 5组文内配图）
   - 每组 prompt 附带中文说明

完成后告知用户所有文件已生成。

## 质量检查

每个阶段输出前，检查该阶段对应的质量项：

阶段二检查：
- 素材审查是否已完成并与用户确认？
- 文章是否删除了所有「水」内容？
- 追问是否保留了价值但去掉了「无知感」？
- 文章是否有足够的信息密度？
- 文章是否完全以作者视角写作？
- 多条对话线索是否真正融合？
- 关键概念、核心术语是否被明确点出？
- 有说服力的具体例子是否被保留？
- 每个核心论点是否至少配了一个场景化例子？
- 段落结构是否正常？（段落体，不是金句体）
- 是否完成了逐句精简审查？（虚词、口水连接词、冗余过渡句、重复修饰是否已清理？）
- 全文是否已清除所有句末句号？（这是最常遗漏的格式问题，必须逐段确认）
- 文件开头是否附带了元信息（字数、主线、段落结构）？
- 犀利度检查（核心原则第 4 条）：
  - 开头是否直接判断读者现状，而非从背景趋势描述入手？
  - 每个核心论点是否包含明确的、有攻击性的判断，而非仅描述机制？
  - 是否直接点名了大多数人在做什么错事？
  - 是否包含 2-3 个试金石式判断标准，让读者无法回避自我归类？
  - 案例是否同时覆盖正面（创造性冲击力）和反面（直接揭示错误行为）？
  - 章节标题是否为叙事句而非学术标签？
  - 收尾是否为二元判断或尖锐事实，而非温和邀请或弱比喻？
  - 密集分析段落是否已拆短（每段 2-4 句）？

阶段三检查：
- 标题是否覆盖 10 种心理触发维度，每种 5 个，共 50 个？
- 标题是否符合通用规则（字数、数字格式、关键词前置、标点策略）？
- 标题多样性是否达标（句式打散、长短交替、开头词去重、语气混搭、视角轮换、情绪光谱、关键词分配）？
- 标题是否通过避坑清单检查？
- 简介是否覆盖 10 种写法，每个 50-80 字？
- 简介是否与标题形成信息增量，而非重复？
- 简介关键信息是否在前 30 字内？

阶段四检查：
- 配图提示词是否符合风格要求？
- 配图提示词是否提供了多样化的视觉思路？

## 用户反馈处理

### 修改幅度判断（重要）

收到用户修改意见时，必须先判断修改幅度，再决定执行方式：

- 小幅调整（默认）：用户说"调一下""优化一下""加一点""改个措辞"等，或反馈的是局部问题（某段太长、某处缺情绪、某个数字要模糊化）。执行方式：使用 Edit 工具做定点修改，只动需要改的句子和段落，其余文字一字不动。禁止整段重写或重新生成全文。
- 大幅调整：用户明确说"重写""推翻重来""整体结构不对"等。执行方式：可以使用 Write 工具重新生成全文。

判断不确定时，默认按小幅调整处理。宁可改少了让用户追加要求，也不要改多了让用户花时间逐字对比。

如果用户反馈「不够精彩」：回到原始对话找被遗漏的精彩内容，检查是否过度精简丢失了细节和案例

如果用户反馈「太长/太啰嗦」：检查是否保留了不必要的追问，检查是否有重复表达需要合并

如果用户反馈「太干/太无聊」：增加故事性元素和对比冲突，用认知沉淀型结构重写，强化作者视角的判断感

如果用户反馈「不够犀利/观点不鲜明」：按核心原则第 4 条逐项对照检查——开头是否在判断、论点是否有攻击性、是否点名了错误行为、是否有试金石判断、收尾是否够狠。通常需要：把描述性语句改写为判断性语句，把温和表达替换为尖锐表达，增加正反案例对比，收尾换成二元判断
